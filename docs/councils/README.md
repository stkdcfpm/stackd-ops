# Council Log

Records of LLM Council sessions run on Stackd Ops decisions.

Each session is saved as `YYYY-MM-DD-topic.md`. The verdict structure is standardised:
- **Where the Council Agrees** — high-confidence signals (multiple advisors converged independently)
- **Where the Council Clashes** — genuine disagreements with both sides presented
- **Blind Spots Caught** — things that only emerged through the peer review round
- **The Recommendation** — a clear, direct call
- **The One Thing to Do First** — a single concrete next step
- **Action Status** — what was done after the verdict

The LLM Council methodology is in `.claude/commands/llm-council.md`.
Trigger a new session with "council this", "war room this", or "pressure-test this".
